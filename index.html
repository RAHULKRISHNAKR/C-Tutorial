Hour 1: Functions & Strings (The Organizers)

Part 1: Functions - The 'Specialist Workers'

Why Use Functions?

The Manager Analogy

Think of your main() function as the 'Manager' of a company. If the manager tries to do all the work (accounting, marketing, sales), the office becomes a mess.

A Function is a 'Specialist Worker' (like an 'Accountant'). The manager's job is to call the specialist to get a specific job done. This keeps the manager's office (main) clean, and you can 'reuse' that specialist many times.

The 3 Parts of a Function

1. The Prototype (The Manager's Contact List)

Place this at the top of your code. It tells the compiler, "Hey, a specialist named add exists. Trust me."

int add(int, int); // Function Prototype
2. The Function Call (The Manager's Request)

This happens inside main. The manager is 'calling' the specialist for help.

// Inside main()...
int sum = add(5, 3); // Function Call
printf("The result is: %d\n", sum);
3. The Function Definition (The Specialist's Instructions)

This is the code inside the function. It's the specialist's 'how-to' manual.

// This can go after main()...
int add(int a, int b) { // Function Definition
  int result = a + b;
  return result; // The specialist gives the 'result' back to the manager
}

Key Concept: 'Pass-by-Value'

When you call add(5, 3), you are giving the function copies of your data. The function can't change your original variables. It's like giving your accountant a photocopy of your documents, not the originals.

Part 2: Strings - The 'Character Train'

What is a String in C?

The Train Analogy

C has no 'string' data type. A string is just an array of characters that ends with a special 'end of train' car: the null terminator (\0).

So, the string "HELLO" is stored in memory like this: ['H'] ['E'] ['L'] ['L'] ['O'] ['\0']

Declaring, Reading, and Printing Strings
#include <stdio.h>

int main() {
  // Create a track for a 19-letter train + the end car
  char name[20];

  printf("Enter your name: ");
  
  // No & is needed! The array name 'name' *is* the address.
  scanf("%s", name);

  printf("Hello, %s!\n", name);
  return 0;
}
Note

To do complex things like find a string's length (strlen()) or compare two strings (strcmp()), you must #include <string.h>. This gives you more 'specialist workers' for strings.

Hour 2: Structures (The 'Data Organizers')

Part 1: The Problem We Need to Solve

How would you store data for a single student? You'd need separate, unconnected variables. This is messy, as all this data belongs together.

The 'Messy' Way
int rollNo = 101;
char name[50] = "Allen";
float gpa = 8.5;

Part 2: The Solution - struct (The 'Bento Box')

Step 1: Create the Blueprint

The Blueprint Analogy

A struct is a blueprint for your own custom data type. You are telling C: "I'm inventing a new type called Student that holds three different items (an int, a char array, and a float) all in one box."

The struct Blueprint (Goes outside main)
struct Student {
  int rollNo;
  char name[50];
  float gpa;
};

Step 2: Use the Blueprint to Create a Variable

Now that you have the blueprint, you can create actual 'bento boxes' from it.

Creating a struct Variable (Goes inside main)
struct Student s1; // 's1' is a variable of our new 'Student' type.

Step 3: Access the Data with the Dot (.) Operator

The dot is how you 'look inside' a specific compartment of the bento box.

Accessing Members
#include <string.h> // We need this for strcpy!

// ... (struct definition and main function start) ...

s1.rollNo = 101;
s1.gpa = 8.5;

// You can't use '=' for strings! You must use strcpy (string copy).
strcpy(s1.name, "Binu");

printf("Student's Name: %s\n", s1.name);
printf("Student's GPA: %.1f\n", s1.gpa);

Part 3: Array of Structures (The 'Shelf' for Bento Boxes)

What about 60 students? You don't make 60 different variables. You make an array of your new Student type.

The Shelf Analogy

An array of structures is like a 'shelf' that holds 60 of your 'student bento boxes'.

Creating an Array of Structs
// Create a 'shelf' that can hold 60 student structs
struct Student cse_class[60];

// Accessing the *first* student (at index 0)
cse_class[0].rollNo = 1;
strcpy(cse_class[0].name, "First Student");
cse_class[0].gpa = 9.0;

printf("The first student is: %s\n", cse_class[0].name);

Hour 3: Pointers (The 'Most Feared' Topic)

Part 1: The Concept - The 'House Address' Analogy

The 'House Address' Analogy

This is the most important concept to understand.

  1. Every variable you create (like int x = 10;) is stored in a 'house' in the computer's memory.
  2. That 'house' has a unique address (e.g., 4000).
  3. A Pointer is a special variable that doesn't store a value (like 10). It stores an address (like 4000).

A pointer is just a variable that holds a memory address.

Part 2: The Two Magic Operators: & and *

1. & (The 'Address-Of' Operator)

Think of & as 'the address of'.
x is the value inside the house (10).
&x is the address of the house (e.g., 4000).

2. * (The 'Value-At-Address' Operator)

Think of * as 'the value at'.
If ptr holds the address 4000, then *ptr means: 'Go to address 4000 and get the value inside that house.' (which is 10).

The 'Aha!' Moment Code
int x = 10;      // A 'house' named 'x' with '10' inside.
int *ptr;      // A pointer, ready to hold an address.

ptr = &x;      // Store the *address of x* in *ptr*.

// --- Let's see what we have --- 
printf("Value of x:     %d\n", x);     // Prints 10
printf("Address of x:   %p\n", &x);    // Prints the address (e.g., 4000)
printf("Value in ptr:   %p\n", ptr);   // Prints the same address (e.g., 4000)
printf("Value at *ptr:  %d\n", *ptr);  // Prints 10

// --- Let's change x *using* the pointer --- 
*ptr = 25;     // 'Go to the address in ptr, and put 25 in that house.'

printf("New value of x: %d\n", x); // This will print 25!

Part 3: The 'Why' - Why Pointers are SO Useful

Problem: Functions only get copies (Pass-by-Value).

Remember? If you send x to a function, it gets a copy. It can't change your original x. But what if you want it to? (Like a swap function).

Solution: Pass the address (Pass-by-Reference).

Instead of sending a copy of x, you send the address of x (using &x). The function can then use a pointer (*ptr) to reach back into main's 'house' and change your original variable.

The Mystery of scanf is Solved!

This is exactly why scanf needs the &! When you write scanf("%d", &age);, you are NOT passing the value of age. You ARE passing the address of the 'age' house. scanf then uses a pointer to go to that exact address in memory and put the number you typed inside.
